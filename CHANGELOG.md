## v0.2.0

### ✨ 新增功能
- **多温度查询支持**：命令行工具新增支持一次性查询多个温度值的功能
- **多温度结果展示**：新增 `print_multi_temp_table` 函数，以紧凑表格形式展示多温度查询结果

### 📝 文档完善
- 在 README.md 中新增详细的命令行使用说明
- 添加了单温度查询、多温度查询、交互式模式等使用示例
- 更新了参数说明表格

### 📦 依赖调整
- **移除 openpyxl 依赖**：精简项目依赖，降低安装复杂度

## v0.1.9

### ✨ 新增功能
- 支持传入numpy数组进行物性参数计算
- 新增对`temp`参数传入numpy数组的支持，实现批量计算功能

### ⚡ 优化改进
- 优化了内部计算逻辑，提取单值计算方法 `_prop_single` 以复用代码
- 对于动力粘度单位转换逻辑进行了统一处理

## v0.1.8

### ✨ 新增功能
- 在 EGASP 类中新增 `concentration_type_to_chinese` 静态方法，用于将浓度类型符号转换为中文名称，提升日志和提示的可读性

### ♻️ 重构与优化
- 优化插值逻辑，当数据缺失时返回 None 而非直接退出程序，并增加详细警告日志
- 改进 `_error_exit` 方法，支持可选错误信息参数，避免空消息记录
- 更新 `eg_fb` 数据表中的浓度值格式，统一使用小数形式表示质量浓度和体积浓度
- 在打印表格时添加对 None 值的处理，显示为 "N/A"，提高输出稳定性

### 🐛 问题修复
- 修复导热系数数据中的个别数值错误

## v0.1.7

## ⚡♻️ 重构与优化
- **统一动力粘度单位处理**：将动力粘度（`mu`）的单位转换逻辑整合到插值返回前统一处理，避免重复转换

## v0.1.6

### ✨ 新增功能
- 调整浓度输入格式为小数形式，范围为 0.1 ~ 0.9，默认值为 0.5
- 更新命令行参数 `-qv` 的默认值和范围说明
- 更新交互式输入提示与默认值

### ♻️ 重构与优化
- 修改核心类 EGASP 中浓度范围、步长和插值逻辑以适配小数形式
- 调整输出表格中质量浓度和体积浓度的数值计算，确保正确显示为百分比

### 📝 文档完善
- 更新文档字符串和错误提示信息，明确使用小数形式
- 同步更新帮助文本中的描述，确保用户理解新的输入范围

### ⚠️ 破坏性变更
- 浓度输入格式从百分比（%）形式调整为小数形式（如 0.5 表示 50%）
- 用户需更新调用方式和输入值以适应新的格式要求

## v0.1.5

### ⚡ ♻️ 重构与优化
- **核心类名重构**：将核心类 `EG_ASP_Core` 重命名为 EGASP，提高命名的简洁性和一致性
- **方法命名优化**：
  - get_egasp → props
  - get_props → prop
  - get_fb_props → fb_props
- **代码结构优化**：改进整体代码组织结构，提升可读性和维护性

### 📝 文档完善
- **完善方法文档字符串**：为所有公共方法添加详细的 docstring，包括参数说明、返回值和异常处理描述
- **增强参数说明**：提供更清晰的参数类型和用途说明
- **提升用户体验**：通过完善文档提高库的易用性和可理解性

### ⚠️ 破坏性变更
- 此版本包含非兼容性更新，需要所有调用方同步修改函数引用
- 建议用户更新代码以适应新的类名和方法名

```python
# 旧版本用法
from egasp.egasp_core import EG_ASP_Core
core = EG_ASP_Core()
result = core.get_egasp(...)

# 新版本用法
from egasp.core import EGASP
core = EGASP()
result = core.props(...)
```
